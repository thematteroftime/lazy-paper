# v1.4 Content Quality — Design Spec

> **Status**: design-approved, awaiting writing-plans handoff. Brainstormed and validated 2026-05-20.
>
> **Tagline**: paper2md v1.4 turns the pipeline into a *single-paper deep-research agent*. PaperDB (KG + retrieval index) becomes the substrate; section composers become tool-using agents that query it; a reviewer stage verifies output. Reference family: Onyx / AI-Scientist-v2 / DeepScientist / claude-code internal patterns — all scoped down to "one paper → one structured 15-section deep-analysis document."

## 1. Goal

Close three content-quality defects observed in v1.3.3 (per `docs/v1_4_roadmap.md`):

1. **Template-driven hallucination** — yang2025 ch01 fabricated "8.6 J/cm³ at η=85%" in a paper that has no energy-storage numbers.
2. **Quoted-symbol drift** — meng2024 conflated E_b=348 kV/cm with test-field=340 kV/cm.
3. **Missed direct source facts** — meng2024 ch10 said "synthesis not addressed" while source explicitly stated "tape-casting".

Each defect maps to a different root cause: lack of typed entity grounding (1), no post-generation source check (2), keyword-matched excerpts missing relevant text (3). Closing all three requires the four-step architecture below.

## 2. Mental model

```
PaperDB (built once per paper, immutable artifact)
  ├─ paper_kg.parquet           closed-schema entity/relation graph
  └─ retrieval.parquet          parent-child chunks + embeddings + BM25 index

Section composer (15 instances per paper, tool-using agents)
  ├─ Tool: query_kg(type, filter)         → typed entities
  ├─ Tool: retrieve(query, top_k)         → evidence chunks with char-span
  ├─ Tool: check_source(claim)            → {found: bool, span: ...}
  └─ Tool: emit_section(draft)            → terminal call, ends loop

Reviewer (runs after composer emits)
  ├─ Tier 1: Python regex (free, deterministic)
  │     - numeric / Fig.N / chem-formula → grep source via KG
  │     - unit normalization
  └─ Tier 2: LLM review (only if Tier 1 flags issues)
        - dimensional scoring: quote-fidelity / grounding / synthesis-depth (1-4)
        - one revision round max
```

The whole shape mirrors Onyx (agent-search + reranker + citation), AI-Scientist-v2 (reviewer stage), and claude-code's tool-call loop. Each pattern is borrowed; **no framework is imported** (LangGraph rejected — patterns reproducible in ~350 LOC of plain Python).

## 5. Data flow

### v1.3.4 (additive, no agent loop yet)

```
s06_context/runner.py
    1. existing context extraction → context.yaml
    2. NEW: kg_extract.build_paper_kg() → paper_kg.parquet
            failure → write kg_extract.failed marker, return None

s07_figure_analyze/runner.py (unchanged)

s08_section_compose/runner.py
    for each section:
        NEW: if Retriever can be built:
                 evidence = retriever.retrieve(section.guidance, top_k=8,
                                               entity_boost=section_entities_from_kg)
             else:
                 evidence = keyword_excerpts(...)        # v1.3.3 fallback
                 log "[degraded] keyword fallback for {paper}"
        composed = llm.chat(compose_prompt(section, evidence))
        NEW: flags = reviewer.regex_check(composed, ...)
        NEW: write critic_flags.yaml + log [critic] flag count
        write chapter md (no revision in v1.3.4 — observe-only)
```

### v1.4.0 (agent loop + revise)

```
s08_section_compose/runner.py
    for each section:
        composed = agent.run(
              section=section,
              kg=Kg(paper_kg.parquet),
              retriever=Retriever(retrieval.parquet),
              prior_bullet=last_section.bullet,
              max_iters=8,
        )
        flags = reviewer.regex_check(composed, ...)
        if flags:
              composed = reviewer.llm_review(composed, flags, evidence)
              flags2 = reviewer.regex_check(composed, ...)
              if flags2:
                    log_unresolved(flags2)   # soft-accept
        findings.append(verified_claims_from(composed))
        write chapter md
```

## 6. Tool contracts (v1.4.0 section agent)

Tools are OpenAI-function-calling style. Each has a JSON schema and pure-function semantics.

```python
query_kg(entity_type: str, filter: dict = None) → list[dict]
    """Return entities of the given type, optionally filtered.
    Types: material / dopant / parameter / value / unit / figure / table / claim / method / comparator.
    Each entity dict has: id, surface_form, normalized, source_span."""

retrieve(query: str, top_k: int = 8, entity_boost: list[str] = None) → list[dict]
    """Return chunks (text + char_span) ranked by hybrid dense+BM25 with optional
    entity-id boost. Top-k clamped to 12."""

check_source(claim: str, expected_value: str = None) → dict
    """Substring-search source for the claim. With expected_value, also runs unit-
    normalize comparison. Returns {found: bool, span: (doc, start, end), evidence: str}."""

emit_section(draft: str) → None
    """Terminal call. Validates draft has at least N citation markers, then writes.
    If validation fails, agent must call retrieve() again."""
```

Agent loop: standard tool-use cycle. Max 8 iterations. If max hit, force `emit_section` with current draft (logged as degraded).

## 7. Reviewer contract

```python
class Flag:
    span: tuple[int, int]       # offset in draft
    claim: str                  # the offending claim
    problem: str                # one of {numeric_not_in_source, fig_not_in_yaml,
                                #         formula_not_in_kg, unit_mismatch}
    evidence: str | None        # what source actually has, if any

def regex_check(draft, source_docs, kg, fig_yaml) → list[Flag]:
    """Deterministic pass. ~free. Catches:
       - numeric claims that don't substring-match (after unit normalize)
       - 'Fig. N' references where N not in fig_yaml
       - chemical formulas not in kg.materials / kg.dopants
       Returns empty list if clean."""

def llm_review(draft, flags, evidence) → str:
    """Gated by `len(flags) > 0`. Single revision round. Prompt: 'Here is your
    draft. Here are flagged issues with evidence excerpts. Revise the flagged
    spans only — do not rewrite passing content.' Returns revised draft."""
```

## 8. Error handling — summary

| Surface | Failure | Behavior |
|---|---|---|
| KG extract | API down / yaml parse fails | Marker file `kg_extract.failed`; s08 uses keyword fallback; pipeline completes |
| Retrieval build | embedding API down | Marker `retrieval.failed`; s08 uses keyword fallback |
| Section agent | iter cap hit | Force `emit_section`; log `[degraded] iter cap @ section X` |
| Section agent | LLM error mid-loop | Retry once; if still fails, fall back to v1.3.3 prompt-stuffing for that section |
| Reviewer regex | fatal Python error (rare) | Skip, accept draft; log |
| Reviewer LLM | LLM error twice | Accept original draft; write unresolved flags to `critic_flags.yaml` |

Universal rule: **soft-degrade per-paper, per-section. Never abort the pipeline.** Output quality drops to v1.3.3 baseline for degraded units; other units unaffected.

## 9. Testing strategy

### v1.3.4 (~12 tests)

```
stages/s06_context/tests/test_kg_extract.py
    - happy path: 5 entities extracted from fixture chapter
    - malformed LLM yaml → marker written, returns None
    - source_span preserved for each entity (regex-recovery code path)

tests/test_retriever.py
    - build_index from 3 chunks → parquet has 3 rows
    - retrieve top_k=2 returns top 2 by cosine score
    - entity_boost re-ranks matched chunks higher
    - hybrid score fusion math (0.6 dense + 0.4 bm25)

stages/s08_section_compose/tests/test_reviewer.py
    - regex_check: source has "8.6 J/cm³", draft says "8.6 J/cm³" → no flag
    - regex_check: source has "8.5 J/cm³", draft says "8.6 J/cm³" → flag
    - unit normalize: "4 MV/cm" == "4000 kV/cm" (no flag)
    - "Fig. 99" not in figures.yaml → flag
    - "Pb²⁺" in kg.materials → no flag; "Au⁺" missing → flag

stages/s08_section_compose/tests/test_units.py
    - kV/cm ↔ kV·cm⁻¹ ↔ MV/cm
    - J/cm³ ↔ J·cm⁻³
    - °C ↔ K (offset 273.15)
```

### v1.4.0 (~6 tests)

```
stages/s08_section_compose/tests/test_section_agent.py
    - happy path: 3 tool calls → emit_section → returns draft with ≥2 citations
    - iter cap at 8 → force emit + degraded log
    - retrieval-only mode (no kg available) → emits using retrieve() alone
    - tool call to bogus tool name → agent recovers in next iter

stages/s08_section_compose/tests/test_reviewer.py (additions)
    - llm_review called only when regex_check flags > 0
    - revised draft passes regex_check second time → returns revised
    - revised still flags → returns original + logs unresolved
```

### Integration — corpus-level regression

After each version (v1.3.4 → v1.4.0):

1. Re-render 10-paper corpus with `--only s08,s09 --force`.
2. `scripts/audit_pptx.py` issue count must stay ≤ current baseline (3 false positives).
3. Truncation rate must stay ≤ 5% (current 2.9%).
4. **New metric: citation density** = % of section bullets carrying at least one `[span:...]` marker:
   - v1.3.4 target: ≥ 60%
   - v1.4.0 target: ≥ 85%

### Manual quality regression

Pick the three known-defect cases from `docs/v1_4_roadmap.md`:

- **yang2025 ch01** — must no longer fabricate "8.6 J/cm³". Step 1+4 close this; v1.3.4 already catches with regex flag, v1.4.0 fixes via agent grounding.
- **meng2024 ch10** — must mention "tape-casting" (source has it). Step 2 closes this via better retrieval.
- **meng2024 ch01** — must quote "348 kV/cm", not "340 kV/cm". Step 4a (regex unit-normalize) closes this.

## 11. Roll-back strategy

- Each release is its own git tag (`v1.3.4`, `v1.4.0`).
- The PaperDB files (`paper_kg.parquet`, `retrieval.parquet`) are per-run-directory; deleting them returns s08 to keyword-fallback mode on the next run.
- Hard rollback: `git checkout v1.3.3` returns to known-good pre-v1.4 state.
- Soft rollback: keep code on v1.4.0 but delete PaperDB files for affected papers; runs auto-degrade to v1.3.3-equivalent behavior.

## 12. Deferred to v1.5

- **Findings Memory consumer** (DeepScientist pattern): cross-section claim accumulator that informs the next section. v1.4 writes the ledger; v1.5 reads it. Defers because we want to validate the basic agent first before adding cross-section state.
- **Multi-agent debate** (Sakana / STORM pattern): 2-3 reviewer personas vote. Premature until we know what single-reviewer regression looks like.
- **DSPy Signature scaffold**: replace string-template prompts with typed Signatures. Maintenance win, not quality win — defer to v2.0.

## 13. Open questions for the writing-plans phase

1. Embedding cost ceiling — should we cap retrieval index size at e.g. 50K tokens of source text to keep DashScope embedding budget predictable?
2. Test fixture for KG extractor — hand-curated mini-paper or LLM-generated synthetic? Hand-curated is more honest but slower.
3. Should v1.3.4 ship with regex-critic-only as `v1.3.4-pre` (1-day) AND PaperDB + retriever as `v1.3.4-main` (4-day) — two micro-releases instead of one?

These are tactical decisions for the implementation plan to make; out of scope for this spec.

## 15. What this spec is NOT

- **Not a re-architecture of the 9-stage pipeline.** STAGE_ORDER stays 9. s06 gets KG as a sub-step; s08 gets agent + reviewer. Nothing else moves.
- **Not a framework adoption.** No LangGraph, no DSPy, no AutoGen, no LangChain. All patterns reproduced in plain Python (~350 LOC total).
- **Not a quality miracle.** We expect to close the three named defects; we do **not** claim to eliminate all hallucination or guarantee SOTA synthesis. Critic tier surfaces residual issues; doesn't promise zero.
- **Not a license risk.** No code lifted from AI-Scientist-v2 (RAIL); patterns described in this spec are conceptual references.
