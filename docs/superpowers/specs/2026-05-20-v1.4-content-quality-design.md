# lazy-paper v1.4 — Content Quality Design Spec

> lazy-paper v1.4 把单篇论文流水线升级为 *单论文深度研究 agent*：在 v1.3.3 不变的提取链路之上，新增 PaperDB 数据层与工具调用 + 审查的章节合成层。**~185 LOC 自写代码** + 5 个成熟 OSS 库（llama-index / bm25s / instructor / pydantic-ai-slim / openai）+ 516 LOC vendored citation 处理器（Onyx MIT）。

## 1. 目标

闭合 v1.3.3 在 10 篇 corpus 实测中观察到的三类内容缺陷：

1. **模板驱动幻觉** — yang2025 ch01 编造 `8.6 J/cm³ at η=85%`，源文无任何储能数据。
2. **符号上下文漂移** — meng2024 把 `test field=340 kV/cm` 误当作 `E_b=340`（源文 E_b 实际 348）。
3. **漏抓直接事实** — meng2024 ch10 写"未提合成方法"，源文明确 "tape-casting"。

三者共因：s08 没有结构化的论文知识，也没有"写完复查"的能力。

## 2. 心智模型

```
PaperDB                            (每篇论文构建一次，库主导，不可变)
  ├─ paper_kg.parquet              10 类闭包实体/关系图（instructor 抽取）
  └─ retrieval.parquet             父子分块 + dense vec + BM25
                                   (llama-index + bm25s + RRF 融合)

Section composer                   (每篇 ~15 个，工具调用 agent)
  ├─ pydantic-ai Agent             循环逻辑由库提供
  └─ 4 个 @agent.tool
       • query_kg     (~10 LOC)
       • retrieve     (~8 LOC)
       • check_source (~8 LOC)
       • emit_section (~4 LOC，终止符)

Reviewer                           (composer 出稿后)
  ├─ T1 regex (确定性，免费)        命中数值/Fig.N/化学式 → 在源文与 KG 中核对
  └─ T2 instructor + Pydantic       T1 触发时才跑；最多重写一次
        dimensional score: quote-fidelity / grounding / synthesis-depth (1–4)
```

**架构原则**：库能做的就用库；自写代码只做项目特有的胶水。

## 8. 工具契约（v1.4.0）

```python
@agent.tool
async def query_kg(ctx: RunContext[SectionContext], entity_type: str,
                   filter: dict | None = None) -> list[dict]:
    """Return entities of the given type, optionally filtered.
    Types: material / dopant / parameter / value / unit / figure / table /
           claim / method / comparator."""
    return ctx.deps.kg.query(entity_type=entity_type, filter=filter)

@agent.tool
async def retrieve(ctx, query: str, top_k: int = 8,
                   entity_boost: list[str] | None = None) -> list[dict]:
    """Hybrid dense + BM25 ranked chunks; top_k clamped to 12."""
    chunks = ctx.deps.retriever.retrieve(query, top_k=min(top_k, 12),
                                         entity_boost=entity_boost or [])
    return [c.to_dict() for c in chunks]

@agent.tool
async def check_source(ctx, claim: str, expected_value: str | None = None) -> dict:
    """Substring + unit-normalized lookup. Returns {found, span, evidence}."""
    return ctx.deps.retriever.check_claim(claim, expected_value)

@agent.tool
async def emit_section(ctx, draft: str) -> str:
    """Terminal call; ends tool loop."""
    if not _has_min_citations(draft, n=1):
        raise ToolValidationError("draft needs ≥1 [span:...] citation marker")
    return draft
```

## 10. 错误处理

| Surface | 失败 | 行为 |
|---|---|---|
| KG 抽取 | instructor 两次解析失败 | 写 `kg_extract.failed`；s08 走关键词回退 |
| 检索索引 | embedding API 不可用 | 写 `retrieval.failed`；s08 走关键词回退 |
| pydantic-ai agent | 命中 iter cap | 库抛 `UnexpectedEndOfStream`；强制对最后部分草稿调用 `emit_section` |
| pydantic-ai agent | LLM 中途错 | 库自动 `ModelRetry` 一次；仍失败则该 section 回退 legacy compose |
| Reviewer regex | 罕见 Python 异常 | 跳过、接受草稿、记录日志 |
| Reviewer LLM | instructor 重试后仍 parse 失败 | 接受原稿，写未解决 flags |

**通用规则**：单篇/单节软降级，流水线绝不中断。
